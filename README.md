# 本项目实现了一个全双工语音交互机器人，具备Agent 智能体思考能力。它不仅能进行语音对话，还能自主调用工具（查天气、查知识库、写报告、数学计算），并支持基于 MD5 指纹的增量式 RAG 知识库构建。

核心特性
# 1. 智能体大脑
自主思考：基于 LangChain ReAct 架构，能够分析用户意图，自动选择工具。

工具调用：内置 RAG 检索、天气查询、外部数据获取、MCP等工具，支持扩展。

中间件 (Middleware)：实现了 AOP 切面编程，支持工具调用的日志监控、动态 Prompt 切换（如进入报告生成模式）。

# 2. 增量式 RAG 知识库
智能去重：利用文件 MD5 指纹技术，只构建新增或变更的文件，彻底解决重复构建和高昂的 Token 成本问题。

多格式支持：原生支持 .txt, .md, .pdf, .docx, .pptx 格式文件的解析与入库。

持久化存储：使用 ChromaDB 进行向量存储，支持元数据过滤。

# 3. 全双工语音交互
ASR (语音识别)：实时 WebSocket 流式接收音频，对接阿里云 Paraformer 模型。

TTS (语音合成)：支持流式语音合成，低延迟响应。

唤醒词检测：支持关键词语音唤醒（如“小仁”），支持打断机制。

# 4. 工程架构
工厂模式：统一管理 LLM 和 Embedding 模型的初始化，解耦具体厂商。

分层设计：清晰分离 接口层、业务层、RAG 层、Agent 层、基础设施层。
 
# 项目结构
🛠技术栈
语言: Python 3.11 (推荐版本，兼容向量库依赖)

Web 框架: FastAPI, Uvicorn

AI 编排: LangChain

向量库: ChromaDB

云服务: 阿里云 DashScope (通义千问), 阿里云 NLS (智能语音交互)

包管理: Uv(极速 Python 包管理器)

开发指南
如何添加新工具？
在 agent/tools/agent_tools.py 中定义一个新的 @tool 函数，tool_list 中注册该工具
在 mcp_server.py 中直接定义工具就可以(该文件需要运行,方可使用)


如何切换模型？
修改 config/model.py 或 .env 配置。工厂模式 (model/factory.py)，业务代码无需变动即可切换不同厂商的 LLM。
